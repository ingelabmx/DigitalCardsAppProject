Tengo este proyecto web existente DigitalCardsApp. Es una aplicación para registrar usuarios y generar tarjetas digitales de lealtad por negocio, con sellos digitales. El punto crítico del sistema es que debe generar tarjetas compatibles con Google Wallet y Apple Wallet, dependiendo del tipo de celular del cliente. 

Quiero iniciar una migración ordenada usando las skills:

- aspnet-core
- playwright

Objetivo principal:
Analiza completamente el proyecto actual y genera el contexto técnico necesario para empezar una migración a ASP.NET Core moderno, sin romper el flujo actual.

Contexto funcional del negocio:
El flujo esperado del producto es:

1. Un cliente se registra en una tarjeta de lealtad de un negocio.
2. El sistema guarda la relación cliente-negocio-tarjeta.
3. Al cliente le llega un correo.
4. En el correo, el cliente puede elegir qué tarjeta necesita:
   - Apple Wallet para iPhone.
   - Google Wallet para Android.
5. El sistema debe generar dinámicamente la tarjeta correcta.
6. La tarjeta debe reflejar los sellos digitales del cliente.
7. Cuando el negocio agrega sellos, la tarjeta debe poder actualizarse.
