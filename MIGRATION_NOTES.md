Esta fase agrega contexto documental y un esqueleto ASP.NET Core paralelo. No reemplaza todavia la aplicacion Web Forms. MySQL HostGator, Google Wallet real, SMTP real, Apple Wallet `.pkpass` y Apple Wallet updates por Web Service/APNs quedan disponibles por configuracion local controlada; los fakes siguen siendo el default para desarrollo, CI y Playwright. El flujo moderno de negocio ahora usa cookie auth, ya no depende de `businessId` visible en URL/formularios, migra passwords de negocio a hashes modernos sin modificar la columna legacy, empieza la operacion diaria desde `/Business/Cards`, usa links Wallet opacos para correos nuevos, registra auditoria de sellos en `StampLedger`, permite que el admin legacy habilite negocios piloto desde `/Admin/Businesses`, registra negocios nuevos desde `/Admin/CreateBusiness` con compatibilidad legacy, administra datos/reset de contrasena desde `/Admin/BusinessProfile/{businessId}`, permite enviar invitaciones seguras a negocios usando el flujo de reset de contrasena sin passwords planos, permite desactivar negocios para bloquear login moderno sin delete fisico, permite gestionar admins desde `/Admin/AdminUsers` sin registro publico, retira la allowlist de clientes del flujo operativo para que el piloto se controle solo por negocio, agrega login/dashboard moderno de cliente con cookie `DigitalCards.Client`, migra passwords de cliente a `ModernClientCredential` con cambio de contrasena moderno, mejora el dashboard cliente con perfil/resumen/estados Wallet y paridad visual legacy, agrega `/Client/Profile` para editar datos basicos del cliente sin cambiar username, convierte `/Business/Dashboard` en resumen operativo con tarjetas recientes y StampLedger, agrega `/Business/Reports` como reporte read-only por negocio autenticado, agrega branding basico de negocio para correos, Wallet, landing publica y dashboard cliente, permite refrescar Wallets ya emitidas despues de cambios de branding desde admin o negocio, centraliza plantillas de correo para Wallet, bienvenida, reset y alertas internas, agrega reset password por email con tokens opacos one-time para clientes y negocios, agrega `/Admin/Support` como diagnostico seguro exportable de clientes, negocios, tarjetas, Wallets, StampLedger y LegacyWalletSync con filtros operativos, export CSV/JSON y reintento manual seguro de updates Wallet, agrega `/Admin/Reports` como reporte operativo minimo read-only, documenta la operacion estable de `app.puntelio.com` con scripts de arranque/health en `ops/windows`, define el reemplazo gradual de Web Forms por negocio con gates y rollback, agrega estado formal de activacion por negocio para distinguir `LegacyOnly`, `PilotModern`, `ModernPrimary`, `LegacyRetired` e `Inactive`, muestra guardrails visibles para negocios `LegacyRetired` en admin y soporte, aplica primeros guards en Web Forms para bloquear negocios `LegacyRetired` y avisar a `ModernPrimary`, agrega smokes manuales de cutover por negocio con fakes o providers reales, permite subir logos publicos por negocio para UI, correo, Google Wallet y Apple Wallet cuando el asset es PNG, documenta el inventario visual Web Forms para guiar la paridad de UI en Razor Pages, agrega un shell autenticado con sidebar/header/footer estilo Web Forms, ajusta las pantallas admin y negocio a una presentacion mas cercana al dashboard legacy, habilita self-service de branding Wallet para negocios sin exponer controles de admin, agrega checklist operativo para cutover real por negocio, mejora los scripts Windows para operar app/tunnel en background con PID, logs, status, stop y restart, bloquea rutas auxiliares de desarrollo/diagnostico para que no queden publicas en el ambiente real, apaga por default la compatibilidad con links Wallet legacy basados en `CardID`, agrega `/Admin/Cutover` para revisar readiness y mover negocios entre estados operativos, muestra estado in-memory de `LegacyWalletSync` en soporte/cutover, agrega links publicos opacos por negocio para registro directo en `/Enroll/{businessToken}`, permite generar QR SVG de registro desde `/Business/Dashboard`, registra reintentos `AdminRetry` para soporte Wallet, y formaliza el runbook final para mover negocios a `ModernPrimary` con rollback por negocio.
