Esta fase agrega contexto documental y un esqueleto ASP.NET Core paralelo. No reemplaza todavia la aplicacion Web Forms. MySQL HostGator, Google Wallet real, SMTP real, Apple Wallet `.pkpass` y Apple Wallet updates por Web Service/APNs quedan disponibles por configuracion local controlada; los fakes siguen siendo el default para desarrollo, CI y Playwright. El flujo moderno de negocio ahora usa cookie auth, ya no depende de `businessId` visible en URL/formularios, migra passwords de negocio a hashes modernos sin modificar la columna legacy, empieza la operacion diaria desde `/Business/Cards`, usa links Wallet opacos para correos nuevos, registra auditoria de sellos en `StampLedger`, permite que el admin legacy habilite negocios piloto desde `/Admin/Businesses`, registra negocios nuevos desde `/Admin/CreateBusiness` con compatibilidad legacy, administra datos/reset de contrasena desde `/Admin/BusinessProfile/{businessId}`, permite gestionar admins desde `/Admin/AdminUsers` sin registro publico, mantiene `/Admin/Clients` como guardrail temporal, agrega login/dashboard moderno de cliente con cookie `DigitalCards.Client`, migra passwords de cliente a `ModernClientCredential` con cambio de contrasena moderno, mejora el dashboard cliente con perfil/resumen/estados Wallet, convierte `/Business/Dashboard` en resumen operativo con tarjetas recientes y StampLedger, agrega branding basico de negocio para correos, Wallet y dashboard cliente, centraliza plantillas de correo para Wallet, bienvenida, reset y alertas internas, y agrega reset password por email con tokens opacos one-time para clientes y negocios.
